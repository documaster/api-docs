Classification System and Class

Classification System

Classification System (Klassifikasjonssystem) is an entity used for grouping Classes (Klasse) which are related to a single concept from the business domain of the archived data. Classification Systems and Classes are the primary means to add comprehensive labeling to folders in the archive.

  • Entity name in the API : Klassifikasjonssystem.
  • Being a core Noark5 model entity, Classification System (Klassifikasjonssystem) is created/modified using the Transaction :: POST /transaction operation.
  • Classification System (Klassifikasjonssystem) can have multiple Classes (Klasse) related to them.
  • Classification System (Klassifikasjonssystem) are associated to Series (Arkivdel) as either primary (main classifcation system) or secondary classification systems. This way Classes (Klasse) from the Classification Systems (Klassifikasjonssystem) can be used to lable/tag folders (Saksmappe,Mappe, Moetemappe) under the Series (Arkivdel)

Class

Class (Klasse) is an entity representing a single concept instance part of a Classification System (Klassifikasjonssystem). Classes are classification criterias which greatly facilitates data retrieval and provides the required data structuring. Classes (Klasse) are associated to folders (Saksmapppe, Mappe, Moetomappe) or records (Journalpost, Basisregistrering, Moeteregistrering) and act as labels for those entities.

  • Entity name in the API : Klasse.
  • Being a core Noark5 model entity, Class (Klasse) is created/modified using the Transaction :: POST /transaction operation.
  • Class (Klasse) can be part of only single Classification System (Klassifikasjonssystem).
  • Classes can be defined in a hierarchy as specified in the Noark5 specification, but only Classes without children classes can be linked to folders/records.
  • Class (Klasse) are uniquely identified by the klasseIdent field, still when making changes/lining classes the id of the class is used.

Working with Classification System and Class

Creating Class with BSM values

Classes (Klasse) are created with the POST /transaction operation. Important unique data field for the Classes (Klasse) is klasseIdent. The sample below also adds sample BSM (business specific metadata) values to the class.

The BSM values associated with the Class are visible in the user interface when the class linked to folder (Saksmappe, Mappe, etc.)

  • transaction::save - new Class (Klasse) with BSM field qsid (Quality System ID) defined in BSM group qs1 (Quality System 1)
  • transaction::link - link the new Class (Klasse) to the existing Classification System (Klassifikasjonssystem).

As a result of the transaction new Class (Klasse) is created with ID = 3168047

...

Content-Type: application/json
X-Documaster-Error-Response-Type: application/json
